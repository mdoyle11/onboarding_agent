Codebase Simplification Summary
===============================
Branch: refactor/remove-langgraph
Date: 2026-04-01

OVERVIEW
--------
The repository has been simplified around a single-container LangChain-based
runtime with deterministic workflow execution for operational paths. The
cleanup removed legacy LangGraph-era indirection, narrowed module ownership,
and reduced duplicated logic across Teams memory, workbook access, payload
normalization, and state persistence.

This was not purely cosmetic refactoring. The resulting codebase is simpler,
but it also includes targeted behavior improvements:

- deterministic adaptive-card actions for send-email and send-DocuSign paths
- thread-scoped Teams memory for channel reply chains
- rehydration of expired channel threads from durable seed context
- bounded retention for card state and thread seed context in Cosmos
- clearer separation between domain logic, workbook integrations, and Graph auth


MAJOR STRUCTURAL CHANGES
------------------------

1. Replaced the old workbook utility hub with narrower modules

   The prior graph_workbook.py file had become a mixed utility module holding:
   - Graph auth
   - workbook/table helpers
   - tracker/roster schema data
   - identity helpers
   - formatting helpers

   It has now been split into clearer boundaries:

   - src/onboarding_agent/integrations/graph/auth.py
   - src/onboarding_agent/integrations/workbook/client.py
   - src/onboarding_agent/integrations/workbook/helpers.py
   - src/onboarding_agent/integrations/workbook/schema.py
   - src/onboarding_agent/integrations/workbook/tracker_client.py
   - src/onboarding_agent/integrations/workbook/staff_roster_client.py
   - src/onboarding_agent/domain/identity.py
   - src/onboarding_agent/domain/formatting.py

   This makes external-system code live under integrations/* and shared
   business primitives live under domain/*.


2. Centralized shared MCP client factories

   Thin client factories were consolidated into:

   - src/onboarding_agent/mcp_server/clients.py

   This reduced repeated client-construction helpers across the MCP tool files.


3. Centralized session-context fields

   Session-context allowlisting previously lived in more than one place and
   could drift. It is now defined in:

   - src/onboarding_agent/agent/session_context.py

   Both the Teams memory layer and the agent runner use the same field list.


4. Centralized payload extraction helpers

   Repeated payload-key normalization logic was moved into:

   - src/onboarding_agent/runtime/payloads.py

   This reduces duplication between runtime/webhooks.py and runtime/jobs.py
   and makes future webhook payload changes safer.


BEHAVIORAL SIMPLIFICATIONS
--------------------------

5. Adaptive-card button actions are now deterministic

   The old hybrid path that translated card clicks into plain-text prompts and
   sent them back through the LLM has been removed for the supported button set.

   Card actions such as:
   - send welcome email
   - send offer letter
   - add to staff roster

   now use deterministic handlers directly in the app. Natural-language Teams
   requests still use the LangChain agent.

   Key files:
   - src/onboarding_agent/integrations/teams/bot.py
   - src/onboarding_agent/integrations/teams/card_actions.py
   - src/onboarding_agent/mcp_server/tools_email.py


6. Teams channel context is now thread-scoped

   Channel conversations previously risked context bleed because memory was too
   conversation-centric. The current implementation scopes memory to:

   - channel:<conversation_id>:thread:<root_message_id>

   The first proactive post seeds structured context for that root thread, and
   replies in that thread reuse it.

   Additional refinement:
   - if the 7-day thread memory expires, the app creates a new session and
     rehydrates it from durable thread seed context tied to the same root post

   Key file:
   - src/onboarding_agent/integrations/teams/memory.py


7. Runtime state is now bounded by explicit lifecycles

   The repo previously had some durable state with no retention policy.
   Current lifecycle behavior:

   - channel thread/session memory: 7 days
   - thread seed context: 30 days
   - adaptive card state: 30 days
   - channel conversation reference: indefinite

   TTL support is implemented in code through the state-store layer and mapped
   to Cosmos per-document TTL.

   Key files:
   - src/onboarding_agent/runtime/state_store.py
   - src/onboarding_agent/runtime/state_store_cosmos.py
   - src/onboarding_agent/integrations/card_state.py
   - src/onboarding_agent/integrations/teams/memory.py


8. Workflow policies remain centralized

   Non-new-hire workflow stage exclusions live in one place instead of being
   scattered through handlers:

   - src/onboarding_agent/domains/onboard/policies.py

   This keeps deterministic workflow behavior explicit and easier to extend.


REMOVED OR REDUCED LEGACY SURFACE
---------------------------------

9. Removed obsolete LangGraph removal artifacts

   The migration rationale now lives in the architecture documentation and in
   commit history. Old one-off migration summaries were removed.


10. Removed dead card-action LLM fallback path

   The card-click-to-text-command path no longer exists for supported
   adaptive-card actions. This reduced branching in Teams message handling and
   removed a class of ambiguity bugs.


11. Removed the old top-level workbook client locations

   The old module locations:
   - src/onboarding_agent/integrations/tracker_client.py
   - src/onboarding_agent/integrations/staff_roster_client.py

   were moved into integrations/workbook/ to reflect actual ownership.
