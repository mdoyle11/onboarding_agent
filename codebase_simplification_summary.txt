Codebase Simplification Summary
================================
Branch: refactor/remove-langgraph
Date: 2026-04-01

OVERVIEW
--------
A series of refactors to reduce code duplication, consolidate shared logic,
and remove obsolete files. All changes are behavior-preserving -- no
functionality was added, removed, or altered. All 68 unit tests pass with
linter (ruff) and type checker (mypy) clean after each change.


CHANGES
-------

1. Consolidated triplicated date formatting function

   An identical ~18-line date formatting function existed in three places:
   - integrations/tracker_client.py  (_format_tracker_date)
   - mcp_server/tools_tracker.py     (_format_stage_date)
   - mcp_server/tools_onboarding.py  (_format_stage_date)

   Extracted a single `format_date()` into integrations/graph_workbook.py.
   All three call sites now import from there.

   Files changed:
   - src/onboarding_agent/integrations/graph_workbook.py
   - src/onboarding_agent/integrations/tracker_client.py
   - src/onboarding_agent/mcp_server/tools_tracker.py
   - src/onboarding_agent/mcp_server/tools_onboarding.py


2. Centralized MCP client factory functions

   Six one-liner factory functions (e.g. `def _tracker() -> TrackerClient`)
   were scattered across mcp_server/tools_*.py modules. Created a single
   mcp_server/clients.py module with all client factories. Each tool module
   now imports from there.

   Files changed:
   - src/onboarding_agent/mcp_server/clients.py  (new)
   - src/onboarding_agent/mcp_server/tools_tracker.py
   - src/onboarding_agent/mcp_server/tools_staff_roster.py
   - src/onboarding_agent/mcp_server/tools_onboarding.py
   - src/onboarding_agent/mcp_server/tools_teams.py
   - src/onboarding_agent/mcp_server/tools_email.py
   - src/onboarding_agent/mcp_server/tools_docusign.py
   - tests/unit/test_mcp_tools.py  (updated patch target)


3. Deleted obsolete migration artifacts

   Two files documented the completed LangGraph removal. The commit history
   and ARCHITECTURE.md serve as the living record.

   Files removed:
   - langgraph_removal_summary.txt
   - docs/proposals/remove-langgraph.md
   - docs/proposals/  (empty directory)
   - docs/  (empty directory)


4. Consolidated identity key normalization

   Identical `_normalize_identity_part()` and composite key-building
   functions existed in both card_state.py and tracker_client.py. Extracted
   `normalize_identity_part()` and `identity_key()` into graph_workbook.py.
   Both modules now import from there.

   Files changed:
   - src/onboarding_agent/integrations/graph_workbook.py
   - src/onboarding_agent/integrations/card_state.py
   - src/onboarding_agent/integrations/tracker_client.py


5. Consolidated DocuSign envelope search logic

   _check_draft_exists_sync() and _find_latest_envelope_for_employee_sync()
   in docusign_client.py were ~65% identical -- same API call pattern, same
   recipient matching, same identity field validation. Extracted a shared
   _search_envelopes_sync() helper. The two public methods are now thin
   wrappers (~10 lines each) that call the shared helper with different
   folder/count/status parameters and shape the return dict. Removed ~60
   lines of duplicated iteration logic.

   Files changed:
   - src/onboarding_agent/integrations/docusign_client.py


6. Adaptive card action builder helper

   Three action button constructions in adaptive_cards.py (email, docusign,
   roster) each had ~15 lines of near-identical if/else logic to build
   enabled vs disabled Action.Submit dicts. Extracted a single
   `_action_button()` helper. Collapsed ~60 lines into three one-liner calls.

   Files changed:
   - src/onboarding_agent/integrations/adaptive_cards.py


7. Extracted shared job helpers in jobs.py

   _process_new_hire_submission() and _process_non_new_hire_submission()
   duplicated tracker lookup/add logic (~20 lines each) and DocuSign draft
   check/create logic (~22 lines each). Extracted:
   - _ensure_tracker_record()  -- lookup or create tracker row
   - _ensure_docusign_draft()  -- check for draft or create one
   Both submission handlers now call these shared helpers.

   Files changed:
   - src/onboarding_agent/runtime/jobs.py


8. Shared Graph API token utility

   The pattern of creating a ClientSecretCredential, acquiring a token for
   https://graph.microsoft.com/.default, and closing the credential was
   repeated in four places. Extracted graph_access_token() as a module-level
   async function in graph_workbook.py. Updated three client modules to use
   it, removing their duplicate credential lifecycle management.

   Files changed:
   - src/onboarding_agent/integrations/graph_workbook.py
   - src/onboarding_agent/integrations/outlook_email_client.py
   - src/onboarding_agent/integrations/forms_client.py
   - src/onboarding_agent/integrations/teams/messenger.py


9. Consolidated card update logic in card_actions.py

   _update_new_hire_card() and _update_docusign_status_card() had identical
   Activity/Attachment creation and context.update_activity() logic (~20
   lines each). Extracted _update_card_via_context() as a shared helper.
   Both are now thin wrappers that build the card dict and delegate.

   Also extracted _ACTION_TO_TOOL mapping dict, replacing duplicated
   if/elif chains in complete_card_action() and
   complete_card_action_without_context().

   Files changed:
   - src/onboarding_agent/integrations/teams/card_actions.py
